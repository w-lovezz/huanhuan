PS：挂载（Mount）的原理
 在 Linux 系统中，挂载是将一个文件系统（如分区、设备或目录）连接到现有的目录树上，使得该文件系统的内容在目录树的某个点（挂载点）上可见。挂载操作会覆盖挂载点原本的内容，使得新挂载的文件系统内容显示在该挂载点上。

Ring3劫持lib库

 LD_PRELOAD是Linux系统的一个环境变量，它可以影响程序的运行时的链接（Runtime linker），它允许你定义在程序运行前优先加载的动态链接库。这个功能主要就是用来有选择性的载入不同动态链接库中的相同函数。通过这个环境变量，我们可以在主程序和其动态链接库的中间加载别的动态链接库，甚至覆盖正常的函数库，那么我们可以使用这种方式实现隐藏进程的目的。

Ring1劫持VFS

 内核态隐藏进程的方法复杂但种类繁多，劫持VFS也算是一种老生常谈的技术，那么什么是VFS呢？
 VFS（Virtual File System，虚拟文件系统）是操作系统内核中的一个抽象层，用于提供对不同文件系统的统一访问接口。它允许应用程序通过一致的系统调用接口访问&不同类型的文件系统，而不需要关心底层文件系统的具体实现细节。

 VFS 的主要功能

  1. 抽象文件系统：提供一个统一的接口，使得不同的文件系统（如 ext4、NTFS、FAT 等）在接口层面表现一致。应用程序可以通过标准的系统调用（如 open、read、write 等）访问文件，而不需要知道底层文件系统的类型。
  2. 文件系统挂载和管理：管理系统中所有已挂载的文件系统。VFS 维护一个挂载点列表，使得文件系统可以挂载到目录树的不同位置。应用程序通过挂载点访问不同的文件系统。
  3. 路径解析：将用户提供的文件路径解析为具体的文件系统操作。VFS 解析路径名并定位到具体的文件或目录，从而调用底层文件系统的具体操作函数。
  4. 缓存和优化：提供文件系统缓存，以提高文件系统操作的性能。VFS 可以缓存文件系统的元数据和文件内容，减少对底层存储设备的访问次数。

 而/proc目录就是一个VFS的具体实现，/proc目录是Linux中的一个虚拟文件系统，它由内核动态创建，用于提供系统和进程的运行时信息。/proc文件系统并不存储实际的数据，而是内核通过VFS接口动态生成内容。当用户态通过cat、ls等命令来访问/proc目录中的信息时，VFS会调用不同的内核函数来处理这些请求；

使用 cat 命令直接读取进程目录下的节点文件时，会调用 lookup系列函数。这个函数用于查找并打开指定的文件。
使用 ls 命令列举所有进程目录时，VFS 会调用readdir系列函数。这些函数用于读取目录的内容，并返回目录中的文件和子目录列表。

 但是列举 /proc 目录和列举某个具体进程目录的实现有所不同，会由proc_root_readdir 函数负责列出 /proc 目录下的所有文件和目录，包括进程目录和静态目录。但无论是读取进程信息还是列举进程目录，最终都会调用proc_pid_instantiate 函数来实例化进程目录。这个函数负责根据当前系统中的进程状态动态生成进程目录项。

—fb，目录与帧缓冲设备（framebuffer devices）相关，提供帧缓冲设备的信息。
